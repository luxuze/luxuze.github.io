# 搬砖经验总结

## 如何让团队的代码保持优雅

1. **保持简单为第一原则**
2. **让工具大于文档和个人约定**
3. **做一件事情只有一种方式**
4. **不需要才是最佳架构**

## 如何提升

1. **深入思考的能力**
   * 经验很重要，因为只有经历过才能更深入的理解痛点以及表面繁华背后需要填的坑，
   * 比经验更重要的是要具备深入思考的能力，时常对从事的专题领域的进行经验总结，才能跳出本位看全局，不断提升自己的判断和分析的能力。
2. **结构化思维方法**
   * 梳理目标，清理目标与自己或将来发展的关系；
   * 做现状分析，知道自己有什么、缺什么以及分析与目标的差距，眼下首要的任务是什么；
   * 路线与措施分析，制定保证有效执行的方案；
   * 下定决心以较高的执行力来实践；
   * 沟能力，在执行决策前，先与分析相关干系人的诉求，并协调其合作。
